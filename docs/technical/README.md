# Technical Documentation

Этот раздел предназначен для профессионального описания проекта как инфраструктурного портфолио.

Целевая аудитория: технические специалисты, работодатели, коллеги и люди, оценивающие архитектуру проекта.

В этом разделе фиксируются:

- архитектура решения;
- логика сетевой топологии;
- схема маршрутизации;
- изоляция сервисов;
- DNS-фильтрация;
- VPN и selective routing;
- anti-DPI routing как технический механизм обработки отдельных типов трафика;
- безопасность;
- резервное копирование;
- мониторинг;
- ограничения и риски.

Документация в этом разделе должна быть написана техническим языком и показывать инженерную логику решений.
