# 09. Anti-DPI Routing

Этот документ описывает общую логику anti-DPI обработки для отдельных типов трафика.

Цель этапа: обрабатывать YouTube и Discord отдельно от VPN, чтобы сохранить низкую задержку и не перегружать VPN-канал видеотрафиком.

## 1. Что такое DPI

DPI — Deep Packet Inspection. Это технология анализа сетевого трафика на стороне провайдера или оператора сети.

DPI может анализировать:

- DNS-запросы;
- SNI;
- TLS ClientHello;
- QUIC / HTTP3;
- IP-адреса CDN;
- сигнатуры протоколов;
- поведение соединений.

Обычная смена DNS не всегда помогает, потому что блокировка может происходить не только по DNS.

## 2. Роль anti-DPI в проекте

В этом проекте anti-DPI используется только для выбранного трафика:

```text
YouTube -> anti-DPI -> direct WAN
Discord -> anti-DPI -> direct WAN
```

Остальной трафик не должен проходить через anti-DPI без необходимости.

## 3. Почему YouTube и Discord не через VPN

YouTube и Discord лучше не направлять через VPN на первом этапе, если anti-DPI работает напрямую.

Причины:

- меньше задержка;
- меньше нагрузка на VPS;
- меньше расход VPN-трафика;
- стабильнее Discord voice;
- YouTube не забивает VPN-канал;
- проще сохранить обычную скорость провайдера.

## 4. Общая схема

```text
Home client
    ↓
OpenWrt
    ↓
Traffic decision:
    ├── YouTube/Discord -> anti-DPI -> WAN
    ├── ChatGPT/Instagram -> VPN
    └── regular traffic -> WAN
```

## 5. Где должен работать anti-DPI

Anti-DPI лучше запускать централизованно на OpenWrt, а не на каждом клиентском устройстве.

Такой подход удобнее:

- работает для всей сети;
- не нужно ставить программы на каждый ПК;
- можно обслуживать телефоны, телевизоры и планшеты;
- проще управлять правилами;
- проще документировать конфигурацию.

## 6. Домены для YouTube

Примерный список доменов, которые могут относиться к YouTube:

```text
youtube.com
www.youtube.com
m.youtube.com
youtu.be
googlevideo.com
ytimg.com
ggpht.com
youtubei.googleapis.com
```

Список может меняться и требует проверки по фактической работе.

## 7. Домены для Discord

Примерный список доменов Discord:

```text
discord.com
discord.gg
discordapp.com
discordapp.net
discordcdn.com
discord.media
```

Discord voice может использовать отдельные адреса и UDP-трафик, поэтому проверять нужно не только открытие приложения, но и голосовую связь.

## 8. QUIC / HTTP3

YouTube активно использует QUIC / HTTP3 поверх UDP 443.

Иногда при проблемах с YouTube помогает отключение или блокировка UDP 443 для клиентов, чтобы браузер перешёл на TCP/TLS.

Но это не нужно включать сразу.

Рекомендуемый порядок:

1. Проверить YouTube без дополнительных правил.
2. Проверить anti-DPI.
3. Только при необходимости тестировать ограничение UDP 443.
4. Фиксировать результат в документации.

## 9. Проверка YouTube

Проверять нужно:

- открытие главной страницы;
- поиск;
- запуск видео;
- перемотку;
- качество 1080p/1440p/4K, если нужно;
- работу на телефоне;
- работу на Smart TV, если используется;
- стабильность в течение 10-15 минут.

## 10. Проверка Discord

Проверять нужно:

- запуск приложения;
- авторизацию;
- текстовые каналы;
- загрузку изображений;
- голосовой канал;
- демонстрацию экрана;
- входящие и исходящие соединения;
- ping, jitter и packet loss.

## 11. Что делать, если работает только часть функций

Если YouTube открывается, но видео не грузится:

- проверить домены `googlevideo.com`;
- проверить UDP 443;
- проверить DNS-запросы в AdGuard;
- проверить, не ушёл ли трафик в VPN случайно.

Если Discord открывается, но голос не работает:

- проверить UDP;
- проверить firewall;
- проверить маршрутизацию;
- проверить, не блокируется ли voice traffic;
- проверить packet loss.

## 12. Что не делать

Не рекомендуется:

- включать anti-DPI сразу для всего трафика;
- одновременно менять VPN, DNS и anti-DPI;
- направлять YouTube через VPN без необходимости;
- использовать Windows-only решения как основное сетевое решение для всей домашней сети;
- публиковать реальные рабочие конфиги без очистки.

## 13. Контрольный список

Этап считается успешным, если:

- YouTube работает без VPN;
- Discord работает без VPN;
- Discord voice имеет приемлемую задержку;
- VPN не перегружается видеотрафиком;
- обычные сайты идут напрямую;
- ChatGPT/Instagram при необходимости идут через VPN;
- DNS-запросы диагностируются через AdGuard.

## Итог

Anti-DPI в этом проекте используется как отдельный механизм обработки YouTube/Discord трафика, а не как замена VPN и не как универсальная обработка всей сети.
