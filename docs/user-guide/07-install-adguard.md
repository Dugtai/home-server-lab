# 07. Install AdGuard Home

Этот документ описывает установку AdGuard Home в отдельный LXC-контейнер Proxmox.

Цель этапа: сделать централизованную DNS-фильтрацию для всей домашней сети.

## 1. Роль AdGuard Home

AdGuard Home будет выполнять функции:

- DNS-сервер для домашних устройств;
- блокировка рекламы;
- блокировка трекеров;
- локальные DNS-записи;
- журнал DNS-запросов;
- базовая фильтрация нежелательных доменов.

AdGuard Home не заменяет OpenWrt. OpenWrt отвечает за маршрутизацию, firewall и DHCP, а AdGuard отвечает за DNS.

## 2. Почему отдельный LXC

AdGuard Home лучше вынести в отдельный контейнер:

- проще обновлять;
- проще перезапускать;
- меньше риск сломать роутер;
- сервис изолирован от OpenWrt;
- можно отдельно делать бэкапы.

## 3. Рекомендуемые ресурсы LXC

| Параметр | Значение |
|---|---|
| CT ID | 101 |
| Name | adguard-home |
| Template | Debian 12 |
| CPU | 1 core |
| RAM | 512 MB |
| Disk | 8 GB |
| IP | 192.168.10.3/24 |
| Gateway | 192.168.10.1 |
| DNS during install | 1.1.1.1 or 192.168.10.1 |

## 4. Создать LXC-контейнер

В Proxmox:

```text
Create CT
```

Основные параметры:

```text
Hostname: adguard-home
Password: strong password
Template: Debian 12
Disk: 8 GB
CPU: 1 core
Memory: 512 MB
Network: vmbr1
IPv4: static 192.168.10.3/24
Gateway: 192.168.10.1
```

Контейнер должен быть подключён к LAN bridge `vmbr1`.

## 5. Обновить систему в контейнере

После запуска контейнера открыть console и выполнить:

```bash
apt update
apt upgrade -y
apt install curl ca-certificates -y
```

## 6. Установить AdGuard Home

Официальный установочный скрипт:

```bash
curl -s -S -L https://raw.githubusercontent.com/AdguardTeam/AdGuardHome/master/scripts/install.sh | sh
```

После установки сервис должен запуститься автоматически.

Проверить статус:

```bash
systemctl status AdGuardHome
```

## 7. Первый вход в веб-интерфейс

Открыть в браузере:

```text
http://192.168.10.3:3000
```

На первом запуске мастер настройки попросит выбрать интерфейсы и порты.

Рекомендуемо:

```text
Web interface: 0.0.0.0:80
DNS server: 0.0.0.0:53
```

Если порт 53 занят, нужно проверить, не запущен ли другой DNS-сервис внутри контейнера.

## 8. Upstream DNS

На первом этапе можно использовать несколько upstream DNS-серверов.

Пример:

```text
1.1.1.1
1.0.0.1
8.8.8.8
8.8.4.4
```

Позже можно настроить DNS-over-HTTPS или DNS-over-TLS, если это нужно.

## 9. Настроить OpenWrt DHCP

После настройки AdGuard нужно в OpenWrt указать клиентам DNS-сервер:

```text
192.168.10.3
```

В LuCI:

```text
Network -> Interfaces -> LAN -> DHCP Server -> Advanced Settings -> DHCP-Options
```

Добавить:

```text
6,192.168.10.3
```

Опция `6` означает DNS server для DHCP-клиентов.

## 10. Обновить сеть на клиентах

На Windows:

```cmd
ipconfig /flushdns
ipconfig /release
ipconfig /renew
```

На Linux:

```bash
sudo resolvectl flush-caches
sudo dhclient -r
sudo dhclient
```

На телефоне проще отключить и снова включить Wi-Fi.

## 11. Проверить работу AdGuard

В веб-интерфейсе AdGuard открыть:

```text
Query Log
```

Там должны появиться DNS-запросы от устройств домашней сети.

Проверить:

- открываются сайты;
- запросы видны в AdGuard;
- реклама частично блокируется;
- нет массовых ошибок DNS.

## 12. Локальные DNS-записи

Рекомендуемые локальные имена:

```text
router.home.arpa      -> 192.168.10.1
proxmox.home.arpa     -> 192.168.10.2
adguard.home.arpa     -> 192.168.10.3
bot.home.arpa         -> 192.168.10.4
cloud.home.arpa       -> 192.168.10.5
ap.home.arpa          -> 192.168.10.10
```

Лучше использовать `home.arpa`, а не `.local`, потому что `.local` часто используется mDNS.

## 13. Базовые фильтры

На первом этапе достаточно включить стандартные фильтры AdGuard.

Не нужно сразу подключать десятки списков: это может ломать сайты и усложнять диагностику.

Рекомендуемый подход:

1. Включить базовый фильтр.
2. Проверить работу сайтов.
3. Добавлять новые списки постепенно.
4. При ошибках смотреть Query Log.

## 14. Что не делать

Не рекомендуется:

- ставить AdGuard внутрь OpenWrt на первом этапе;
- сразу включать слишком много фильтров;
- использовать AdGuard как единственную точку отказа без fallback-плана;
- публиковать `AdGuardHome.yaml` в GitHub без очистки;
- открывать веб-интерфейс AdGuard в интернет.

## 15. Контрольный список

Этап считается завершённым, если:

- контейнер AdGuard запущен;
- веб-интерфейс доступен по `192.168.10.3`;
- клиенты получают DNS `192.168.10.3`;
- DNS-запросы видны в Query Log;
- локальные DNS-записи работают;
- интернет у клиентов не сломан.
